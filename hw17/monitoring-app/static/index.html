üî¥ Test Error Rate Alerts

GET /api/force-error
Always returns HTTP 500 Internal Server Error. Perfect for testing error rate alerts in Prometheus/Grafana.
GET /api/users/invalid-id
Randomly returns 400 or 500 errors (5% chance). Simulates realistic error scenarios for alert testing.
GET /api/any-endpoint?error=true
Add ?error=true parameter to any API endpoint to force an error response.

üü° Test Latency Alerts

GET /api/slow?delay=2000
2 second delay response. Tests latency alerts for response time thresholds.
GET /api/slow?delay=5000
5 second delay response. Simulates extremely slow responses for critical latency alerts.
GET /api/any-endpoint?delay=1000
Add ?delay=1000 parameter to simulate 1 second delay on any endpoint.

üü¢ Normal API Endpoints

GET /api/users
User API endpoint with normal response time (100-300ms). Returns JSON data with simulated processing delay.
GET /api/orders
Order management endpoint. Includes random delay and occasional errors for realistic testing.
GET /api/products
Product catalog endpoint. Simulates database queries with variable response times.

‚ö° Quick Test Commands

Use these terminal commands to generate test data:
# Generate multiple errors simultaneously
for i in {1..50}; do curl -s http://localhost:8080/api/force-error > /dev/null & done

# Test latency with custom delay
curl "http://localhost:8080/api/slow?delay=3000"

# Test normal endpoint with parameter
curl "http://localhost:8080/api/users?delay=500"

# Generate mixed traffic (run in separate terminals)
./load-test.sh  # See GitHub for load test script

# Monitor live metrics while testing
watch -n 1 'curl -s http://localhost:8080/metrics | grep http_requests_total'
